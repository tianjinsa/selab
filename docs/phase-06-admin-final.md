# 阶段 6：管理员后台完善与最终验证

## 分支

`phase/06-admin-final`

## 目标

完善管理员后台、Mock 演示数据开关、系统参数、部署构建和最终验收。

## 交付内容

- 管理员用户、任务、帖子、商品、订单、举报、纠纷管理。
- 封禁、禁言、限制发布、扣信用分、恢复信用分。
- 任务分类、商品分类、年级推荐映射、敏感词、知识库、AI 配置维护。
- AI 风险报警、咨询分类统计、工具调用日志。
- 社区热点、词云、AI 总结。
- 任务倾向分析。
- Mock 数据开关，仅影响后台展示类功能。
- 构建脚本：用户端和管理员端产物输出到后端 `public`，Express 静态托管 `/` 与 `/admin`。
- 最终验收脚本和运行说明。

## 用户体验补充

- 后台列表提供搜索、筛选、状态标签和空状态。
- 所有危险操作二次确认，并写入管理员操作日志。
- Mock 开关在页面上明确标注当前数据来源。
- 处理举报/纠纷时提供处理建议和影响说明，减少误操作。
- 最终页面能在演示环境快速切换账号和数据场景。

## 验收点

- 管理员可完成需求文档列出的后台操作。
- Mock 数据只影响后台展示，不改变真实业务。
- 前后端可构建，后端可托管用户端和管理员端。
- 关键接口、WebSocket、权限隔离和业务闭环通过基础验证。
