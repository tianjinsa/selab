# 阶段 2：校园任务互助系统

## 分支

`phase/02-task-system`

## 目标

完成任务发布、模拟支付、浏览筛选、申请接单、交付验收、评价和后台管理闭环。

## 交付内容

- 任务分类、任务列表、任务详情、筛选搜索。
- 任务发布表单、敏感词校验、酬金范围校验、截止时间校验。
- 模拟支付页面和支付流水。
- 任务申请后自动进入私信并发送任务申请卡片。
- 发布者在卡片中同意或拒绝申请。
- 多人申请只能同意一人，其他申请卡片自动失效。
- 接单者提交交付凭证，发布者验收或进入纠纷。
- 超时扫描、退款/结算流水、信用分扣减日志。
- 任务评价、周榜/月榜、任务关键词倾向分析。
- 管理员任务列表、下架、举报、纠纷处理和参数配置。

## 用户体验补充

- 发布前展示费用、截止时间、支付说明和取消后果。
- 支付取消后返回编辑页，再次回退直接退出且不保留草稿。
- 低信用用户在申请入口前就能看到原因，不等到提交才失败。
- 任务状态展示为时间线，减少用户对“下一步该谁操作”的困惑。
- 卡片失效时显示失效原因。

## 验收点

- 用户可发布任务并完成模拟支付。
- 申请任务会自动发送私信卡片。
- 发布者可同意或拒绝，且只能选择一个接单者。
- 任务可提交凭证、验收、完成。
- 信用分低于 4 分不能接任务，低于 6 分有提示。
- 管理员可处理任务举报和纠纷，后台可查看任务倾向分析。
