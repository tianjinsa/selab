# 最终验证说明

## 构建与启动

```bash
npm install
npm run build
npm run start
```

`npm run build` 会把用户端和管理员端构建到 `server/public`，后端 Express 负责静态托管：

- `/`：用户端
- `/admin`：管理员端

## 已验证闭环

- 用户注册、登录、资料维护、忘记密码。
- 管理员独立登录，用户 Token 不能访问管理员接口。
- WebSocket 私信、未读通知、任务/商品卡片状态同步。
- 任务发布、模拟支付、申请接单、同意/拒绝、交付、验收、评价、超时扫描。
- 社区发帖、Tag、瀑布流、评论楼中楼、点赞、收藏、关注、热榜、词云、举报处理。
- 二手商品发布、分类申请、购买申请、模拟担保支付、交付、收货、订单完成、年级推荐。
- AI WebSocket 流式输出、停止生成、知识库工具、公开业务查询工具、咨询分类统计、风险报警。
- 管理员用户、任务、社区、二手、AI、综合审核和 Mock 展示开关。

## SQL Server 与演示模式

系统启动时会检测 SQL Server。如果 `localhost:8887` 的 `CampusLifeService` 可用，则使用 SQL Server 文档集合表持久化数据；否则使用 `server/data/app-data.json` 本地演示数据。

`/api/health` 会返回当前数据模式，便于验收时确认运行环境。
