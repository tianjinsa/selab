# 阶段 1：主系统、认证、个人中心、私信通知基础

## 分支

`phase/01-main-system`

## 目标

完成校园智能生活服务主系统骨架，让普通用户和管理员都能进入各自入口，并具备统一认证、个人中心、私信和通知基础。

## 交付内容

- Node.js + Express 后端骨架。
- SQL Server 连接检测、配置文件、数据访问层和种子数据。
- 用户注册、登录、退出、JWT 登录态保持。
- 学号 12 位校验、手机号唯一、密码加密存储。
- 简单忘记密码流程：无需认证，按学号和手机号重置。
- 管理员 `/admin` 登录，管理员 Token 与用户 Token 分离。
- 用户资料读取与更新，学号不可修改，信用分只读。
- WebSocket 鉴权、在线连接管理。
- 一对一私信、历史消息、未读数、通知中心基础。
- 用户端应用外壳、个人中心、消息中心、通知中心。
- 管理端应用外壳和基础仪表盘入口。

## 用户体验补充

- 注册页即时提示学号、手机号和密码格式问题。
- 登录失败要区分账号不存在、密码错误、账号封禁。
- 个人中心明确展示不可修改字段，避免用户误操作。
- 私信列表显示未读数、最后消息、免打扰状态和空会话引导。
- 通知中心支持筛选、单条已读、全部已读和删除。

## 验收点

- 用户可注册、登录、退出。
- 微信登录不存在。
- 用户可改昵称、手机号、头像、联系方式、简介和密码。
- 学号不可修改。
- 管理员可通过 `/admin` 登录。
- 用户端和管理员端 Token 不互通。
- 两个用户可建立私信会话，消息可持久化并产生未读通知。
