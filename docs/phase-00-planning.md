# 阶段 0：规划与工程基线

## 分支

`phase/00-planning`

## 目标

建立项目的 Git 工作流、阶段拆分文档、工程约束和可持续迭代边界。

## 交付内容

- 初始化 Git 主分支 `main`。
- 添加 `.gitignore`，避免提交本地技能缓存、依赖、构建产物和环境变量。
- 添加 `README.md`，说明阶段分支与开发原则。
- 为后续 6 个子系统阶段建立独立文档。

## 用户体验补充

- 每个阶段都必须包含清晰的空状态、错误提示、确认反馈和返回路径。
- 所有关键流程必须能被课程验收人员直接操作，不依赖口头说明。
- 对需求文档中未明确的异常路径，优先选择不丢数据、可撤销、可追踪的处理方式。

## 验收点

- `main` 分支存在。
- 阶段文档存在并能指导后续开发。
- 后续每个阶段都可独立分支、提交、合并。
