## 一、项目总述
### 1.1 项目背景
为整合校园生活服务资源，解决校园内信息分散、互助效率低、闲置资源流转难等问题，打造一体化校园综合服务生态，规划建设校园智能生活服务平台，覆盖任务互助、社区交流、二手交易、智能咨询等核心校园场景。

### 1.2 项目目标
以「校园智能生活服务系统」为统一底座，搭建用户体系统一、服务入口统一、数据标准统一的综合服务平台，串联校园生活全场景，提升校园信息流通效率与生活便捷度。

### 1.3 开发顺序说明
遵循「底座先行、业务递进、智能升级」的开发原则，按以下优先级分阶段落地：
1. 校园智能生活服务系统（主系统，全平台公共底座）
2. 校园任务互助系统（高频生活服务场景）
3. 校园社区论坛系统（内容互动交流场景）
4. 校园二手市场系统（闲置资源交易场景）
5. 校园信息智能体系统（AI智能增值场景）

#### git说明
每个阶段在单独的分支中进行开发，开发完成后合并至主分支，主分支为稳定版本。
每个阶段中每个任务单独提交git commit，commit信息中包含任务编号与简要描述，便于追踪与回溯。

### 技术栈
- 前端：微信小程序
- 管理员后台页面：Vue3 + Element Plus（所有管理员后端都使用web）
- 后端：node.js + express + sql server

## 二、校园智能生活服务系统（主系统）
### 2.1 系统定位
本系统是整个平台的核心底座与统一入口，承担全平台用户身份管理、基础社交能力、子系统入口聚合、个人数据统一管理的职责，是所有业务子系统的公共支撑层。

### 2.2 功能需求
#### 2.2.1 用户认证模块
- **账号注册登录**：支持学号/手机号注册、密码登录、微信快捷登录（需要管理员生成邀请码来注册，微信快捷注册也需要邀请码，如果没有邀请码则无法注册）；支持校园身份校验对接(无法实现，弄个假的验证)；实现全平台单点登录，一次登录即可访问所有子系统，所有子系统通过统一的认证接口进行访问。
- **账号中心与权限**：支持个人信息管理（头像、昵称、学号、联系方式等）、密码修改、账号安全设置；

#### 2.2.2 私信沟通功能
- **即时聊天私信模块**：支持用户一对一私信沟通，兼容文字、图片消息格式；支持消息已读/未读状态实时展示；支持跨子系统发起会话（如二手交易买卖双方沟通、任务供需双方沟通），聊天记录云端持久化存储。
技术上使用WebSocket实现实时通信，消息存储在后端数据库中，确保消息的可靠传输与持久化。
每对用户间的私信会话在数据库中以唯一会话ID标识，消息记录按时间顺序存储，支持分页加载历史消息。
- **消息通知与提醒**：支持新消息推送提醒，支持消息免打扰设置；支持消息搜索、历史记录查看、消息删除等功能。
- **通用卡片系统**：
  - 支持在私信中发送任务卡片、任务申请结束卡片、帖子卡片（分享帖子）等操作卡片，接收方可直接点击卡片进行任务报名、帖子查看等操作。
  - 系统自动记录操作卡片的交互数据，确保任务流转、交易订单、内容互动等操作的状态一致性。

#### 2.2.3 消息通知模块
- **多类消息统一汇聚**：聚合任务报名提醒、任务状态变更、验收提醒、酬金到账、评论回复提醒、点赞提醒、@提醒、关注更新、系统公告等全场景通知，统一列表展示。
- **通知已读管理**：支持通知分类查看、一键已读、单条标记已读；清晰区分已读/未读状态，支持删除历史通知。

#### 2.2.4 平台ui

登录进入平台后，显示多tab导航，分别为「任务互助」「二手市场」「社区论坛」「智能体」「我的」，点击对应tab进入子系统首页。
初始进入「社区论坛」

### 2.3 非功能需求
- **性能**：登录响应时间≤2s，私信消息发送延迟≤1s；支持万级用户并发访问。
- **安全**：用户密码加密存储，敏感信息脱敏展示；支持异常登录检测与风险提醒。

## 三、校园任务互助系统
### 3.1 系统定位
面向校园用户的互助服务子系统，覆盖代取快递、代办事务、学业互助、技能服务等校园高频互助场景，实现任务发布、接单、交付、结算全流程闭环管理。

### 3.2 功能需求
#### 3.2.1 任务市场模块
- **多类型任务发布**：支持用户发布跑腿代办、学业互助、技能服务、其他互助四大类任务，可填写任务详情、酬金金额、截止时间、交付要求等信息；发布前校验用户身份与信用状态。
- **接单者排行榜**：基于接单完成率、用户好评率、累计接单量等维度，生成周榜/月榜；展示接单用户信用等级与历史评价，为任务发布者提供选择参考。
- **任务浏览与筛选**：支持按任务类型、酬金区间、发布时间、距离等维度筛选；支持关键词精准搜索；任务详情页展示发布者信息、任务要求、报名状态等核心信息。
- **任务流转闭环**：支持用户报名接单（可以进入对方私信进一步沟通），发布者确认最终接单人选；接单者完成任务后提交交付凭证，发布者验收确认后完成酬金结算；支持任务取消、超时自动关闭、请求管理员介入等异常流程。（沟通使用咨询系统）
- **支付系统**：支持微信支付、校园一卡通支付等多种支付方式（支付系统都弄成假的，显示一个支付成功页面），确保任务酬金结算的安全与便捷。
发布任务时，要求发布者支付任务酬金至平台托管账户，任务完成后由平台将酬金支付给接单者；若任务取消或超时未完成，酬金退回发布者账户。

申请任务时，同时通过私信系统向发布者发送报名通知卡片，发布者可在私信中与接单者沟通任务细节.
任务结束时，任意一方可在私信中发起验收与结算操作（发送任务卡片），系统将自动记录任务状态流转与资金结算数据，确保数据一致性。

#### 3.2.2 消息通知模块（使用主系统的）
- **多类消息统一汇聚**：聚合任务报名提醒、任务状态变更、验收提醒、酬金到账、系统公告等全场景通知，统一列表展示。
- **通知已读管理**：支持通知分类查看、一键已读、单条标记已读；清晰区分已读/未读状态，支持删除历史通知。

#### 3.2.3 后台管理模块
- **系统参数配置**：管理员可配置任务分类、酬金上下限、任务超时规则、信用分增减规则等系统参数。
- **用户举报与人工审核**：支持用户对违规任务、违约接单行为提交举报；管理员审核举报内容，可执行下架任务、扣除信用分、封禁账号等处理操作。

### 3.3 非功能需求
- **性能**：任务列表加载时间≤2s，任务发布操作响应时间≤1.5s。
- **可靠性**：任务状态流转与资金结算数据强一致，支持异常状态自动预警。
- **可扩展性**：支持后续新增任务类型、结算规则，无需重构核心流程。

### 3.4 导出部分功能，为智能体系统提供function调用
- **任务信息查询**：提供任务列表、详情查询接口，供智能体系统调用，实现任务智能推荐、描述生成等功能。
- **任务状态查询**：提供任务状态查询接口，供智能体系统调用，实现任务提醒、异常处理等功能。
- ** 用户信用查询**：提供用户信用分、接单记录查询接口，供智能体系统调用，实现信用评估、风险提示等功能。
- **任务搜索与筛选**：提供任务搜索与筛选接口，供智能体系统调用，实现智能问答、个性化推荐等功能。

## 四、校园社区论坛系统
### 4.1 系统定位
校园内容社区与互动交流平台，支持用户发布多元内容、互动讨论、信息分享，打造校园信息交流、社群互动与生活分享的核心阵地。

### 4.2 功能需求
#### 4.2.1 社区内容与互动模块(类似小红书)
- **多类型内容发布**：支持用户发布纯文字、图文、求助、经验分享等类型帖子，支持上传图片、添加话题标签；内容发布触发敏感词校验。
- **智能检索与排行榜**：支持关键词全文检索帖子内容，可按话题、作者、发布时间筛选；设置热门榜、最新榜，基于浏览量、点赞量、评论量综合排序。
- **楼中楼回复**：支持一级评论与楼中楼二级回复，支持评论点赞、举报、@用户功能；@操作触发对应通知。
- **基础互动能力**：支持帖子点赞、收藏、分享；支持关注作者，个人主页展示用户全部发布内容。

#### 4.2.2 消息通知模块（使用主系统的）
- **多类消息统一汇聚**：聚合评论回复提醒、点赞提醒、@提醒、关注更新、系统公告等通知，统一列表展示。
- **通知已读管理**：支持通知分类查看、一键已读、删除历史通知，清晰区分已读/未读状态。

#### 4.2.3 后台管理模块
- **系统参数配置**：管理员可配置社区板块分类、话题标签、内容审核规则、排行榜更新频率等参数。
- **用户举报与人工审核**：支持用户举报违规内容、不当言论；管理员对举报内容与平台巡检内容进行审核，可执行删除帖子、禁言、封禁账号等操作。

### 4.3 非功能需求
- **性能**：帖子列表加载时间≤2s，内容发布响应时间≤1.5s。
- **内容安全**：内置敏感词过滤机制，支持发布前拦截与发布后巡检。
- **并发能力**：支持千级用户同时在线浏览互动，无明显卡顿。

### 4.4 导出部分功能，为智能体系统提供function调用
- **内容检索与推荐**：提供帖子搜索、热门推荐接口，供智能体系统调用，实现个性化推荐、内容生成等功能。
- **查看帖子详情与评论**：提供帖子详情、评论列表接口，供智能体系统调用，实现内容分析等功能。

## 五、校园二手市场系统
### 5.1 系统定位
校园闲置物品交易平台，提供二手商品发布、浏览、交易、售后全流程服务，促进校园内闲置资源高效流转，降低学生消费成本。

### 5.2 功能需求
#### 5.2.1 商品发布与管理模块
- **闲置商品发布模块**：用户可发布二手商品，填写商品名称、分类、价格、成色、详情描述，上传商品图片；支持设置自提/同城配送等交易方式。
- **我的在售商品管理模块**：用户可管理自身发布的商品，支持编辑信息、上下架操作、查看浏览量与咨询数据。
- **商品审核模块**：商品发布后进入审核流程，采用「系统初筛+人工复审」机制，违规商品自动拦截，审核通过后方可前台展示。

#### 5.2.2 商品浏览、搜索、筛选功能
- **首页推荐模块**：首页基于用户浏览偏好、商品热度做个性化推荐；设置热门分类、最新上架等专属专区。
- **分类导航模块**：按数码、服饰、书籍、生活用品等品类设置多级分类导航，支持分类快速筛选。
- **商品详情模块**：完整展示商品信息、卖家信用、咨询入口、收藏/下单按钮；支持商品图片放大查看。（咨询使用私信系统）
- **搜索与筛选**：支持关键词搜索商品；支持按价格区间、成色、发布时间、交易方式等多维度筛选。

#### 5.2.3 交易订单管理功能
- **下单交易模块**：用户选中商品后可创建订单，确认交易信息；支持平台担保交易，买家付款后资金托管，确认收货后打款至卖家；订单全状态可追踪（待付款、待交付、待收货、已完成、已取消）。

#### 5.2.4 举报、投诉、纠纷售后功能
- **订单评价模块**：交易完成后买卖双方可互相评价，评价结果计入用户信用分；支持追加评价与图片评价。
- **商品举报模块**：用户可举报违规商品、虚假描述商品，管理员审核后执行下架、处罚等处理。
- **纠纷仲裁**：交易出现争议时，买卖双方可发起申诉并提交举证；管理员介入仲裁，根据凭证执行退款、赔付等处理。

### 5.3 非功能需求
- **性能**：商品列表加载时间≤2s，商品详情页打开时间≤1.5s。
- **交易安全**：担保交易机制保障资金安全，交易数据全程留痕，支持订单全链路溯源。
- **数据一致性**：商品库存、订单状态、资金流水数据强一致，无超卖、资金差错问题。

### 5.4 导出部分功能，为智能体系统提供function调用
- **商品信息查询**：提供商品列表、详情查询接口，供智能体系统调用，实现商品智能推荐、描述生成等功能。
- **交易状态查询**：提供订单状态查询接口，供智能体系统调用，实现交易提醒、纠纷处理等功能。
- **用户信用查询**：提供用户信用分、评价记录查询接口，供智能体系统调用，实现信用评估、风险提示等功能。
- **商品搜索与筛选**：提供商品搜索与筛选接口，供智能体系统调用，实现智能问答、个性化推荐等功能。

## 六、校园信息智能体系统
### 6.1 系统定位
基于大语言模型的校园智能咨询系统，为全平台提供智能问答、知识查询、工具调用等AI能力，提升平台服务效率与用户咨询体验。

### 6.2 功能需求
#### 6.2.1 知识库模块
- **知识库构建与管理**：录入校园规章制度、办事流程、常见问题、校园资讯等结构化与非结构化知识；支持知识分类管理、词条增删改查、版本迭代更新。
- **语义检索能力**：支持基于自然语言的知识库语义检索，精准匹配用户问题对应的知识条目，为智能问答提供数据支撑。

#### 6.2.2 工具调用模块
- **内置工具集**：集成其他子系统的功能接口，如任务互助、二手市场、社区论坛等，实现智能体对各子系统的能力调用。
- **自动调度能力**：根据用户问题自动识别所需工具，调用对应能力并返回结构化结果。

#### 6.2.3 智能问答模块
- **多场景问答**：支持校园办事咨询、平台功能指引、生活问题解答、学业常识解答等场景；支持多轮对话，可理解上下文语义。
- **答案溯源**：AI回答标注知识来源，支持用户跳转查看原始知识库条目，保障答案可信度。

#### 6.2.4 会话管理模块
- **会话历史留存**：保存用户与智能体的全部对话历史，支持用户查看历史会话、继续历史对话。
- **数据统计**：后台可统计问答量、高频问题、问题解决率等数据，为知识库优化提供数据依据。

#### 6.2.5 提示词工程模块
- **场景化提示词配置**：针对客服咨询、学习辅助、内容生成等不同场景，配置专属系统提示词（不同智能体使用不同的提示词），定义AI回答风格、边界与规则。
- **版本管理**：支持提示词版本迭代、灰度测试、效果对比，持续优化AI回答质量。

### 6.3 非功能需求
- **性能**：问答平均响应时间≤3s，简单常识类问题响应≤1.5s。
- **准确性**：校园场景问题回答准确率≥90%；无法回答的问题明确告知用户，并引导人工咨询渠道。
- **安全性**：严格管控回答边界，禁止输出违规、违法、超出校园场景的内容；对话数据合规存储，保护用户隐私。

### 6.4 提供agent模式来调用其他子系统的function
- **任务互助功能调用**：智能体可调用任务互助系统的任务信息查询、任务状态查询、用户信用查询等接口，实现任务智能推荐、异常提醒等功能。
- **二手市场功能调用**：智能体可调用二手市场系统的商品信息查询、交易状态查询、用户信用查询等接口，实现商品智能推荐、交易提醒等功能。
- **社区论坛功能调用**：智能体可调用社区论坛系统的内容检索与推荐、帖子详情与评论查询等接口，实现个性化内容推荐、内容分析等功能。

## 七、系统对接总要求
1. **统一用户体系**：所有子系统复用主系统用户认证能力，实现单点登录，用户基础数据全平台互通。
2. **统一消息体系**：各子系统通知可汇聚至主系统统一消息中心，同时子系统保留独立消息列表。
3. **统一交互规范**：所有子系统UI风格、交互逻辑与主系统保持一致，通过主系统导航入口进入。
